# 企业画像长期记忆系统设计方案

## 1. 设计目标

### 1.1 核心需求
- 为企业画像agent提供长期记忆能力
- 设计通用记忆管理系统，支持多智能体复用
- 实现"怎么查询的怎么存储"的语义一致性
- 支持通过传参或进程形式供其他智能体使用

### 1.2 设计原则
1. **通用性**：记忆管理系统应独立于具体业务逻辑
2. **可扩展性**：支持多种存储后端（内存、文件、数据库、向量数据库）
3. **语义一致性**：查询接口与存储接口保持一致性
4. **易用性**：提供简单的API供其他智能体集成
5. **性能**：支持高效的记忆检索和存储

## 4. 存储后端设计

### 4.1 存储适配器接口

```python
class StorageBackend(ABC):
    """存储后端抽象基类"""
    
    @abstractmethod
    async def save(self, record: MemoryRecord) -> bool:
        """保存记忆记录"""
        pass
    
    @abstractmethod
    async def get(self, record_id: str) -> Optional[MemoryRecord]:
        """获取记忆记录"""
        pass
    
    @abstractmethod
    async def search(
        self,
        query: Optional[str] = None,
        embedding: Optional[List[float]] = None,
        filters: Optional[Dict[str, Any]] = None,
        limit: int = 10
    ) -> List[MemoryRecord]:
        """搜索记忆记录"""
        pass
    
    @abstractmethod
    async def update(self, record_id: str, updates: Dict[str, Any]) -> bool:
        """更新记忆记录"""
        pass
    
    @abstractmethod
    async def delete(self, record_id: str) -> bool:
        """删除记忆记录"""
        pass
    
    @abstractmethod
    async def count(self, filters: Optional[Dict[str, Any]] = None) -> int:
        """统计记忆记录数量"""
        pass
```

## 6. 集成方案

### 6.1 与企业画像Agent集成

```python
class EnterpriseProfileAgentWithMemory:
    """带长期记忆的企业画像Agent"""
    
    def __init__(
        self,
        memory_manager: LongTermMemoryManager,
        checkpointer=None
    ):
        self.memory_manager = memory_manager
        self.enterprise_memory_manager = EnterpriseMemoryManager(
            memory_manager.storage,
            namespace=("enterprise", "profiles")
        )
        # 原有Agent初始化代码...
    
    async def process_with_memory(self, message: str, config: Dict[str, Any]):
        """带记忆处理的流程"""
        
        # 1. 检索相关记忆
        relevant_memories = await self.memory_manager.search(
            query=message,
            agent_id=self.name,
            session_id=config.get("session_id")
        )
        
        # 2. 整合记忆到上下文
        memory_context = self._build_memory_context(relevant_memories)
        
        # 3. 执行原有逻辑
        response = await self._original_process(message, config)
        
        # 4. 存储新记忆
        if self._should_store_memory(response):
            await self.memory_manager.store(
                content=message,
                metadata={
                    "response": response,
                    "enterprise": config.get("company"),
                    "agent": self.name
                },
                agent_id=self.name,
                session_id=config.get("session_id")
            )
        
        return response
```

## 14. 总结

本设计方案提出了一个通用、可扩展的长期记忆管理系统，专门为企业画像Agent场景优化。系统具有以下特点：

1. **通用性**：通过抽象接口设计，支持多种存储后端和检索策略
2. **语义一致性**：实现了"怎么查询的怎么存储"的设计理念
3. **易用性**：提供简单的API和多种集成方式
4. **高性能**：通过缓存、索引和优化策略确保系统性能
5. **可扩展性**：支持水平和垂直扩展，满足不同规模需求

该系统可以显著提升企业画像Agent的智能水平，通过长期记忆实现更连贯、更个性化的服务，同时为其他智能体提供通用的记忆管理能力。
