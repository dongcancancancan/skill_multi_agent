# 多智能体金融助手项目 - 技术栈分析报告

## 📋 项目概述

**项目名称**: Multi-Agent Financial Assistant  
**项目类型**: 多智能体金融分析系统  
**技术架构**: 基于 LangChain + LangGraph 的多智能体协作框架  
**开发语言**: Python 3.12  
**许可证**: MIT  

这是一个复杂的多智能体金融分析系统，采用现代化的AI技术栈，具备多智能体协作、意图分析、并行执行、状态管理和多种数据源集成等特性。

---

### 1. AI & LLM 框架
- **LangChain (0.3.27)** - 大语言模型应用开发框架
- **LangGraph (0.6.2)** - 多智能体工作流编排和状态管理

## 🔧 核心技术特性

### 1. 多智能体协作架构
- 基于LangGraph的状态图工作流
- 支持串行和并行智能体执行
- 智能体间数据共享和依赖管理
- 动态工具调用和结果聚合

### 2. 意图分析与路由
- **双层意图识别**: LLM意图分析 + 关键词匹配后备方案
- **中文语义分析**: 基于jieba分词的中文语义理解
- **并行工具检测**: 自动识别需要并行执行的工具
- **补充工具机制**: 检测并补充遗漏的工具调用

### 3. 异步与并发处理
- 基于AsyncIO的异步编程模型
- 支持最大8个工具的并行执行
- 工具执行超时控制（160秒）
- 智能重试和快速失败机制

### 4. 状态管理与缓存
- Redis分布式会话管理
- 长期记忆缓存机制
- 会话上下文保持
- 分布式锁确保数据一致性

### 5. 数据源集成
- **金融数据**: AkShare中国金融市场数据
- **向量检索**: Milvus向量数据库支持绿色金融政策搜索
- **关系数据**: MySQL主数据库存储企业基础信息
- **图数据**: Neo4j图数据库支持供应链关系分析
- **外部API**: XML和JSON格式的外部服务集成

### 6. 监控与日志
- Loguru结构化日志系统
- 性能监控和慢查询检测
- LangSmith集成的LLM调用追踪
- 详细的错误处理和异常捕获

## 🐳 Docker化部署方案

项目提供了完整的容器化部署方案，包含以下服务：

### 核心数据服务
- **MySQL 8.0** - 主数据库，存储企业基础信息
- **Redis 7** - 缓存和会话管理，支持分布式锁
- **Neo4j 5.15** - 图数据库，用于供应链关系分析

### 向量数据库
- **Milvus 2.3.3** - 向量数据库，用于绿色金融政策搜索
- **etcd** - Milvus依赖的配置存储
- **MinIO** - Milvus依赖的对象存储

### 管理界面
- **phpMyAdmin** - MySQL管理界面 (http://localhost:8080)
- **Redis Commander** - Redis管理界面 (http://localhost:8081)
- **Neo4j Browser** - 图数据库管理界面 (http://localhost:7474)
- **MinIO Console** - 对象存储管理界面 (http://localhost:9001)

### 快速启动
```bash
# 进入部署目录
cd deploy/docker

# 给脚本执行权限
chmod +x docker-setup.sh

# 启动所有服务
./docker-setup.sh

# 或直接使用docker-compose
docker-compose up -d
```

### 服务端口配置
| 服务 | 端口 | 用途 |
|------|------|------|
| MySQL | 3306 | 主数据库 |
| Redis | 6379 | 缓存和会话 |
| Neo4j (Bolt) | 7687 | 图数据库连接 |
| Neo4j (HTTP) | 7474 | 图数据库管理界面 |
| Milvus | 19530 | 向量数据库 |
| phpMyAdmin | 8080 | MySQL管理界面 |
| Redis Commander | 8081 | Redis管理界面 |
| MinIO | 9000 | 对象存储 |
| MinIO Console | 9001 | 存储管理界面 |
| Milvus UI | 9091 | 向量数据库管理 |

## ⚙️ 配置管理

### 环境变量配置
项目提供了 `deploy/env.template` 模板文件，包含以下配置分类：

#### 数据库配置
```env
# MySQL（主数据库）
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_DATABASE=multi_agent
MYSQL_USER=root
MYSQL_PASSWORD=Password

# Redis（缓存和会话）
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_DB=0

# Neo4j（图数据库）
NEO4J_URI=bolt://localhost:7687
NEO4J_USERNAME=neo4j
NEO4J_PASSWORD=Password

# Milvus（向量数据库）
MILVUS_HOST=localhost
MILVUS_PORT=19530
```

#### AI服务配置
```env
# OpenAI
OPENAI_API_KEY=your_openai_api_key_here
OPENAI_MODEL=gpt-4

# Anthropic
ANTHROPIC_API_KEY=your_anthropic_api_key_here

# LangSmith监控
LANGCHAIN_TRACING_V2=true
LANGCHAIN_API_KEY=your_langsmith_api_key_here
```

### YAML配置系统
- **default.yaml** - 默认配置文件
- 支持环境变量替换：`${ENV_VAR}`
- 支持多环境配置覆盖
- 缓存和性能参数集中管理

## 🚀 项目启动流程

### 1. 环境准备
```bash
# 安装Python依赖
pip install -r requirements.txt

# 或使用Poetry
poetry install

# 复制环境变量模板
cp deploy/env.template .env
# 编辑.env文件，填入实际配置
```

### 2. 启动中间件
```bash
# 进入部署目录并启动Docker服务
cd deploy/docker
./docker-setup.sh
```

### 3. 启动应用服务
```bash
# 启动FastAPI服务
python app/main.py

# 启动Chainlit聊天界面
chainlit run app/ui/chainlit_app.py

# 启动模拟金融服务
python app/server/server.py
```

### 4. 访问应用
- **API文档**: http://localhost:8000/docs
- **聊天界面**: http://localhost:8001  
- **模拟服务**: http://localhost:8082
